Adding Images to Markdown: A Complete Guide

Basic Syntax

In Markdown, you can add images using two methods:

1. Inline Image Syntax

![Alt text](URL)

Where:

  • Alt text is the alternative text description of the image
  • URL is the path to your image

Example:

![My Profile Picture](https://example.com/images/profile.jpg)

2. Reference-Style Syntax

![Alt text][image-id]

[image-id]: URL

Example:

![Profile Picture][profile]

[profile]: https://example.com/images/profile.jpg

Image Paths

Type Example Usage
Absolute URL https://example.com/image.jpg Web-hosted images
Relative Path ./images/picture.jpg Local repository images
Root Relative /images/picture.jpg From repository root

Adding Title Text

You can add a title that appears when hovering over the image:

![Alt text](URL "Image title")

Example:

![Profile Picture](profile.jpg "My Profile Picture from 2023")

Advanced Features

Image Size (HTML)

While pure Markdown doesn't support image sizing, you can use HTML:

<img src="image.jpg" alt="Alt text" width="200" height="100">

Image Alignment

Using HTML and CSS:

<img align="right" src="image.jpg" alt="Alt text">
Note: Not all Markdown processors support HTML. Check your platform's documentation for compatibility.

Best Practices

  • Always provide meaningful alt text for accessibility
  • Use relative paths for repository images
  • Optimize images before adding them
  • Consider image size and loading time
  • Use consistent naming conventions

Common Issues and Solutions

Issue Solution
Image not displaying
  • Check URL/path spelling
  • Verify file exists
  • Ensure proper permissions
Broken image links
  • Use relative paths for local images
  • Verify image hosting service availability
